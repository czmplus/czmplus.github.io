2. Docker容器操作

2.1 基本操作

选项 作用
–name 自定义容器名
-d 将容器放后台启动
-it 分配一个交互式的终端
-p 端口映射
-P 映射随机端口
-v 数据卷映射

容器运行的原则

  1. 一个容器最好只做一件事

  2. 容器运行必须保证容器内PID为1的进程一直存在

PID为1的进程是谁决定的？

  1. docker run运行容器后加的命令

  2. 如果在运行容器时没有加任何命令，则以容器镜像中CMD中的命令作为PID为1的进程

==如果退出容器,那么容器状态会怎么样?==

-it 运行容器,容器是一个系统.

容器退出后,容器关闭了

==关于docker run -it说明==

-i 进入交互模式

-t 分配一个终端(运行命令解释器) /bin/bash 或 /bin/sh

-it 进入容器并有个命令行(终端).

好处运行容器的同时可以进入到容器中,缺点:退出容器则容器结束

容器想要放在后台(-d)一直运行的话,那么容器运行对应初始命令,必须夯住(阻塞/前台运行),否则容器就会退出

==如何进入已经运行中的容器==

通过-it -d + 指定命令解释器持续运行

docker run -itd --name czm6 centos /bin/sh

docker exec小结

  • 用于与已经运行中的容器进行连接.
    • -it 交互模式分配终端
    • docker exec -it --name xxx
    • 镜像名字:版本 参数(一般/bin/bash 或/bin/sh)
    • nginx -t ;nginx -s reload

==exec vs attach区别==

docker container指令 exec attach
共同点 连接到容器 连接到容器
区别 连接的时候创建终端 容器要有终端(容器进程中要有个/bin/bash 或/bin/sh) 只能连接到已有的终端中
区别 不同的exec连接互不影响 共用,所有连接都一样.

==docker run vs docker create,start,stop,restart==

  • docker run背后
    • docker pull #如果镜像不存在.
    • docker create 创建容器.
    • docker start 启动容器.
    • docker stop 关闭容器. #向容器中的主进程,pid 1 进程发出信号(kill),关闭.
    • docker restart 重启容器

==我们传输文件到容器中,如何做?==

  • 之前我们在windows与linux之间通过lrzsz实现.
  • docker container cp === docker cp
源 目标
宿主机——>容器 上传 docker cp 路径或文件 容器:容器中目录
容器——>宿主机 下载 docker cp 容器:容器中目录 路径或文件

docker cp:

  • 宿主机与容器间互传数据.

  • -a表示保存文件属性不变(uid,gid)

2.2 ==commit用于自定义镜像 ⭐⭐⭐⭐⭐==

  • 处理容器,生成镜像.

  • 使用流程:

    • 运行服务镜像或系统镜像,启动后成为容器.
    • 根据我们的需求,对容器进行修改.
    • 测试完成后.
    • 最后通过commit命令把容器保存为镜像.
    • 根据新生成的镜像创建容器并测试

    具体步骤：

    1. 创建并运行容器：
      首先，你需要基于一个基础镜像（如centos:latest）来创建一个容器，并进入该容器的shell环境。例如：

      docker run -it --name my_container centos

      这将会启动一个名为my_container的容器，并docker exec -it <容器名> /bin/sh进入容器的bash shell。

    2. 在容器内进行修改：
      在容器内部，你可以安装新的软件包、更改配置文件、创建新文件或目录等。例如，你可以使用yum命令来安装一个软件包：

      yum install -y <package_name>

      请注意，所有的更改都将在容器的可写层中进行，而基础镜像保持不变。

    3. 提交容器为新镜像：
      一旦你在容器中完成了所有的修改，你可以使用docker commit命令将容器的当前状态保存为一个新的镜像。这个命令的基本格式如下：

      docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

      其中，CONTAINER是你要提交的容器的ID或名称，REPOSITORY和TAG是新的镜像的名称和标签。例如：

      docker commit my_container my_custom_image:latest

      这将创建一个名为my_custom_image，标签为latest的新镜像，其中包含了你在my_container容器中所做的所有更改12。

    4. 查看新镜像：
      使用docker images命令可以查看你所有的镜像，包括你刚刚创建的自定义镜像。

    5. 使用新镜像：
      你现在可以使用这个新的自定义镜像来创建新的容器，并在其中运行你的应用程序。

    注意：虽然docker commit命令可以很方便地创建自定义镜像，但它也有一些缺点。例如，它不会记录镜像的创建历史，因此其他人可能难以理解或维护你的镜像。此外，如果你的容器包含敏感数据或配置信息，那么这些信息也可能被包含在最终的镜像中，从而带来安全风险。因此，在可能的情况下，最好使用Dockerfile来构建镜像，因为它可以保留完整的构建历史，并提供更多的灵活性和安全性

3. 端口映射

  1. 使用docker的时候 外界访问docker容器中的服务或端口,需要使用端口映射.本质类似于iptables防火墙的端口映射.

  2. 应用场景: 未来容器需要被外界访问(80端口).需要暴漏在外界一个端口.

  3. 用户通过端口访问容器中的某个端口.

  4. 实现方法:docker run通过-p选项实现.

  5. 本质是通过iptables nat规则实现的.nat表中创建了docker自定义的链.

3.1 背后操作

  • 添加了对应防火墙规则 iptables -t nat -A DOCKER ! -i docker0 -p tcp -m tcp --dport 80 -j DNAT --to-destination 172.17.0.2:80
  • 开启内核转发功能(需要我们自己开启)

3.3 端口映射案例

1）一对一端口映射

docker run
-p选项(小写字母P) 宿主机端口:容器中的端口 -p 80:80 -p 443:443
-p :容器中端口 -p :80 表示宿主机端口随机,很少用
-p 端口范围:端口范围 80-88:80-88

2） 多端口映射

3）ip绑定端口

  • 显示用户只能通过宿主机的某个网卡连接这个端口，起安全防护作用

小结：

端口映射

• -p 宿主机端口:容器端口

• -p 宿主机IP:宿主机端口:容器端口

• -p 宿主机IP::容器端口(随机端口)

• -p 宿主机端口:容器端口/udp

• -p 宿主机IP::容器端口/udp 使用宿主机的10.0.0.103这个ip地址的随机端口的udp协议映射容

器的udp53端口

• -p 81:80 –p 443:443 可以指定多个-p

• -p 80-88:80-88 端口范围映射
