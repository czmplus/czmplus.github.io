4. Dos

4.1 dos基本介绍

DOS（Disk Operating System）是早期个人计算机上的一类操作系统，由微软公司开发，并主要用于IBM PC及其兼容机。从1981年的MS-DOS 1.0到1995年的MS-DOS 6.22，DOS在长达15年的时间里，作为微软在个人计算机上使用的一个操作系统载体，推出了多个版本，并在IBM PC兼容机市场中占有举足轻重的地位。

DOS系统的主要特点如下：

  1. 命令行操作：DOS系统通过命令行的方式，允许用户直接操纵管理硬盘中的文件。这种操作方式虽然在现代操作系统中已不是主流，但它在DOS系统中仍然是非常基础和重要的部分。
  2. 界面简单：DOS系统的界面十分简单，只有黑白两种颜色，没有图形用户界面（GUI）。所有的操作都要通过键盘输入指令完成。这样的设置使得DOS系统的体积大大减小，运行速度快，让早期的电脑得以快速启动。
  3. 易于自定义：DOS系统用户可以按照自己的需要在硬盘上创建空目录，然后根据自己的想法去添加和删除文件或者文件夹。用户还可以自由配置自己的个性化操作方式。
  4. 系统稳定：由于DOS系统体验十分简单，而且很多操作都是人工干预的，因此它的系统十分稳定。许多老电脑（如486机）依然在使用DOS系统并且运行良好。

尽管随着技术的发展，DOS系统已经被更先进的操作系统（如Windows、macOS、Linux等）所取代，但在某些特定领域或场合（如学习编程、维护老旧设备等），DOS系统仍然具有一定的应用价值和意义

==标识符==

  • 一个名字，本质上是个字符串，用来指代一个值
  • 只能是大小写字母、数字、下划线，也可以是Unicode字符
  • 不能以数字开头
    ·不能是Go语言的关键字
    ·尽量不要使用“预定义标识符”，否则后果难料
    ·大小写敏感

标识符建议:

  1. 不要使用中文
  2. 非必要不要使用拼音
  3. 尽量遵守上面的命名规范，或形成一套行之有效的命名规则

==字面常量==

  • 它是值，不是标识符，但本身就是常量，不能被修改。
    Go语言中，boolean、rune、integer、float、complex、string都是字面常量。其中，rune、integer、float、complex常量被称为数值常量

2.2 常量

go语言中，常量不可以用数组
100 “abc”
这些数值要保存在内存中
a="abc"赋值语句，名称是a，被赋值，a是一个指代，指向某个值如果可以，再次可以赋值，a称为变量，a的指代可以发生变化

  1. 如果可以，再次可以赋值，a称为变量，
    • a的指代可以发生变化,同类型可以再次赋值，a是变量(a=100不可以，a="aaa"可以)
  2. 如果不可以
    • a有类型要求
    • a就是不可以再被赋值，a和第一个赋给它的值之间的联系不可改变，a称为常量标识符

  • 常量:标识符的指向的值一旦建立，不可改变，要求定义是必须建立关联

  • 变量:标识符的指向的值建立后，可以改变

    1. 类型也可以变，往往动态语言Js、Python，举例a=100，a=“abc”，a=[1,2,3]
    2. 同类型可以变

注意:Go语言的常量定义，必须是能在编译器就要完全确定其值，所以，值只能使用字面常量。这和其他语言不同!例如，在其他语言中，可以用常量标识符定义一个数组，因为常量标识符保证数组地址不变，而其内元素可以变化。但是Go根本不允许这样做。

3. 程序控制

程序控制

  • 顺序
    。按照先后顺序一条条执行
    。例如，先洗手，再吃饭，再洗碗
  • 分支
    。根据不同的情况判断，条件满足执行某条件下的语句
    。例如，先洗手，如果饭没有做好，玩游戏:如果饭做好了，就吃饭;如果饭都没有做，叫外卖
  • 循环
    。条件满足就反复执行，不满足就不执行或不再执行
    。例如，先洗手，看饭好了没有，没有好，一会来看一次是否好了，一会儿来看一次，直到饭好了，才可是吃饭。这里循环的条件是饭没有好，饭没有好，就循环的来看饭好了没有

3.8 标准输入

在Go语言中， fmt.Scan, fmt.Scanf, fmt.Scanln 这些函数，它们都属于 fmt 包，用于从标准输入（通常是键盘）读取并解析数据。

  1. fmt.Scan

fmt.Scan 函数从标准输入读取文本，并根据参数的类型和数量解析和存储它们。它返回读取并成功赋值的参数个数以及遇到的任何错误。

  1. fmt.Scanf

4.5 go语言里线性数据结构和哈希表的优劣

在Go语言中，线性数据表（如数组、链表等）和哈希表（在Go中通常通过map实现）是两种常用的数据结构，它们各有优劣，适用于不同的场景。

线性数据表的优劣

优点：

  1. 简单直观：线性数据表的结构简单，容易理解和实现。
  2. 顺序访问：对于需要顺序访问数据的场景，线性数据表非常合适。
  3. 动态扩容：虽然数组的大小在声明时是固定的，但可以通过动态数组（如Go中的slice）来实现动态扩容。链表则天生支持动态扩容。

缺点：

  1. 随机访问效率：数组支持高效的随机访问（时间复杂度为O(1)），但链表不支持，其随机访问的时间复杂度为O(n)。
  2. 内存使用：链表由于需要额外的指针来存储节点之间的关系，因此相对于数组来说，内存使用效率较低。
  3. 操作复杂：在链表中进行插入和删除操作虽然方便，但相对于数组来说，实现起来较为复杂。

哈希表的优劣

优点：

  1. 快速查找：哈希表通过哈希函数将键名映射到哈希值，从而实现了快速的查找、插入和删除操作（平均时间复杂度为O(1)）。
  2. 灵活性：哈希表的键和值可以是任意类型的数据，提供了很大的灵活性。
  3. 自动扩容：在Go中，当map中的元素数量超过其容量时，Go运行时会自动进行扩容，无需手动干预。

缺点：

  1. 哈希冲突：哈希表无法完全避免哈希冲突，当哈希冲突发生时，需要采用一定的冲突解决策略（如拉链法或开放寻址法），这可能会增加查找、插入和删除操作的时间复杂度。
  2. 无序性：哈希表是无序的，即无法保证元素的存储顺序与插入顺序一致。如果需要保持元素的顺序，则需要额外的数据结构来辅助。
  3. 内存使用：哈希表在存储数据时，除了存储键和值本身外，还需要额外的空间来存储哈希值和解决哈希冲突（如链表或开放寻址的额外空间），因此相对于线性数据表来说，内存使用效率可能较低。

总结

在选择使用线性数据表还是哈希表时，需要根据具体的应用场景和需求来权衡。如果需要频繁地进行随机访问操作，且数据量不大，可以考虑使用数组或切片；如果需要保持元素的顺序，且操作频繁，可以考虑使用链表；如果需要快速查找、插入和删除操作，且不关心元素的顺序，则哈希表是一个很好的选择。在实际开发中，也可以根据具体需求将这两种数据结构结合起来使用，以达到最优的性能和效果。

5. Map原理及数据类型

5.1 数据结构

5.1.1 映射

在Go语言中，映射（Map）是一种非常重要的数据结构，它提供了一种将键（Key）映射到值（Value）的方式。映射类似于其他编程语言中的字典（Dictionary）或哈希表（Hashtable），用于存储一系列无序的键值对。

1. 定义和创建

Go语言中的映射是引用类型，它基于键来存储值。映射的键必须是唯一的，并且可以使用==运算符进行比较。映射的创建可以使用字面量或make函数。

  • 使用字面量创建：

    m := map[string]int{"name": 1, "age": 20}

    这里，m是一个映射，其键类型为string，值类型为int，并且初始时包含两个键值对。

  • 使用make函数创建：

    m := make(map[string]int, 10) // 创建一个容量为10的映射

    使用make函数时，第二个参数是可选的，它指定了映射的初始容量。注意，这只是一个容量的建议值，映射在需要时会自动扩容。

2. 访问和修改

映射中的元素可以通过键来访问和修改。

  • 访问元素：

    value := m["name"] // 访问键为"name"的元素

    如果键不存在，将返回该类型的零值（对于int来说是0，对于string来说是空字符串等）。

  • 修改元素：

    m["name"] = 2 // 修改键为"name"的值为2

    如果键不存在，它将被添加到映射中。

3. 遍历

映射可以使用for range循环进行遍历，但遍历的顺序是不确定的，因为映射是无序的。

4. 删除元素

可以使用内置的delete函数从映射中删除键值对。

5. 容量和长度

映射的容量是内部存储桶（bucket）的数组大小，但它不是固定不变的，会随着元素的增加而自动扩容。映射的长度（即键值对的数量）可以使用内置的len函数获取。

6. 注意事项

  • 映射的键必须是可比较的，这意味着键的类型可以是内置类型（如字符串、整数等），也可以是结构体（只要结构体中的字段也都是可比较的）。但是，切片、函数和包含不可比较字段的结构体不能作为映射的键。
  • 映射的键是唯一的，但值可以重复。
  • 映射是无序的，即你不能预测遍历映射时键值对的返回顺序。
  • 如果尝试对未初始化的映射（即nil映射）进行赋值或访问操作，将会导致运行时错误。因此，在使用映射之前，必须确保它已经被初始化。

  2. 什么是冲突的key?什么是相同的key?hash值相同key就相同吗?

    冲突的key

    在哈希表（Hash Table）或类似的数据结构中，冲突的key指的是两个不同的key通过哈希函数计算后得到了相同的哈希值（Hash Value）。由于哈希函数的设计是将无限大的输入集映射到有限大的输出集，因此总会有不同的输入产生相同的输出，即哈希冲突（Hash Collision）。

    相同的key

    相同的key则是指两个key在逻辑上是相等的，即它们代表相同的数据或信息。在Java等编程语言中，如果两个对象作为HashMap的key，并且它们的hashCode()方法返回值相等且equals()方法返回true，则这两个对象被认为是相同的key。

    hash值相同key就相同吗?

    hash值相同并不意味着key就相同。如上所述，哈希冲突的存在意味着不同的key可以映射到相同的哈希值。因此，仅通过哈希值相同来判断两个key是否相同是不准确的。在HashMap等数据结构中，当两个key的哈希值相同时，还需要通过equals()方法来进一步判断这两个key是否真正相等。

    例如，在Java中，如果两个对象的hashCode()方法返回相同的值，但在equals()方法中它们并不相等，那么这两个对象在HashMap中将被视为不同的key，即使它们的哈希值相同。这是因为HashMap在解决哈希冲突时，会首先比较哈希值，如果哈希值相同，则会进一步使用equals()方法来比较key是否相等

常见算法：

  • SHA(Secure Hash Algorithm)安全散列算法，包含一个系列算法，分别是SHA-1、SHA-224、SHA-256、SHA-384 和SHA- 512.
    • 数字签名防篡改
  • MD5(Message Digest Algorithm 5)信息摘要算法5，输出是128位。运算速度叫SHA-1快
    • 用户密码存储
    • 上传、下载文件完整性校验
    • 大的数据的快速比对，例如字段很大，增加一个字段存储该字段的hash值，对比内容开是否修改

6. go函数

函数
数学定义

  • y=f(x)，y是x的函数，x是自变量。y=f(x0,x1,… xn)

Go函数

  • 由若干语句组成的语句块、函数名称、参数列表、返回值构成，它是组织代码的最小单元
  • 完成一定的功能

函数的作用

  1. 结构化编程对代码的最基本的封装，一般按照功能组织一段代码
  2. 封装的目的为了复用，减少冗余代码
  3. 代码更加简洁美观、可读易懂

6.1 返回值

返回值

  • 可以返回0个或多个值
  • 可以在函数定义中写好返回值参数列表
  • 可以没有标识符，只写类型。但是有时候不便于代码阅读，不知道返回参数的含义
  • 可以和形参一样，写标识符和类型来命名返回值变量，相邻类型相同可以合并写
  • 如果返回值参数列表中只有一个返回参数值类型，小括号可以省略

以上2种方式不能混用，也就是返回值参数要么都命名，要么都不要命名

  • return
  • return之后的语句不会执行，函数将结束执行
  • 如果函数无返回值，函数体内根据实际情况使用return
  • return后如果写值，必须写和返回值参数类型和个数一致的数据
  • return后什么都不写那么就使用返回值参数列表中的返回参数的值

6.3 作用域

函数会开辟一个局部作用域，其中定义的标识符仅能在函数之中使用，也称为标识符在函数中的可见范围。这种对标识符约束的可见范围，称为作用域。

1、语句块作用域

if、for、switch等语句中使用短格式定义的变量，可以认为就是该语句块的变量，作用域仅在该语句块中。

swith、select语句中的每个子句都被视为一个隐式的代码块.

2、显式的块作用域

在任何一个大括号中定义的标识符，其作用域只能在这对大括号中。

3、universe块

宇宙块，意思就是全局块，不过是语言内建的。预定义的标识符就在这个全局环境中，因此bo01、int、nil、true、false、iota、append等标识符全局可见，随处可用。

4、包块

每一个package包含该包所有源文件，形成的作用域。有时在包中顶层代码定义标识符，也称为全局标识符。

6.4 匿名函数

顾名思义，就是没有名字的函数。在函数定义中，把名字去掉就可以了，

匿名函数主要作用是用作高阶函数中，它是传入的逻辑。若一个函数允许传入的参数是函数类型，就是把操作逻辑外置。

6.5 递归函数

简单来说，递归就是函数自己调用自己。有2种实现方式，一种是直接在自己函数中调用自己，一种是间接在自己函数中调用的其他函数中调用了自己。

  1. 递归函数需要有边界条件、递归前进段、递归返回段
  2. 递归一定要有边界条件
  3. 当边界条件不满足时，递归前进
  4. 当边界条件满足时，递归返回

6.52 间接递归

间接递归调用，是函数通过别的函数调用了自己，这一样是递归。
只要是递归调用，不管是直接还是间接，都要注意边界返回问题。但是间接递归调用有时候是非常不明显，代码调用复杂时，很难发现出现了递归调用，这是非常危险的。

总结

  1. 递归是一种很自然的表达，符合逻辑思维
  2. 递归相对运行效率低，每一次调用函数都要开辟栈帧
  3. 递归有深度限制，如果递归层次太深，函数连续压栈，栈内存就可能溢出了
  4. 如果是有限次数的递归，可以使用递归调用，或者使用循环代替，循环代码稍微复杂一些，但是只要不是死循环，可以多次迭代直至算出结果
  5. 绝大多数递归，都可以使用循环实现
  6. 即使递归代码很简洁，但是能不用则不用递归

6.7 闭包

自由变量:未在本地作用域中定义的变量。例如定义在内层函数外的外层函数的作用域中的变量。

闭包:就是一个概念，出现在嵌套函数中，指的是内层函数引用到了外层函数的自由变量，就形成了闭包。很多语言都有这个概念，最熟悉就是JavaScript。闭包是运行期动态的概念。

代码分析
第15行调用outer函数并返回inner函数对象，并使用标识符fn记住了它。outer函数执行完了，其栈帧上的局部变量应该释放，包括inner函数，因为它也是局部的。但是，c、inner对应的值都不能释放，因为fn要用。所以这些值不能放在栈上，要放到堆上。在Go语言中，这称为变量逃逸，逃逸到堆上在某个时刻，fn函数调用时，需要用到c，但是其内部没有定义c，它是outer的局部变量，如果这个c早已随着outer的调用而释放，那么fn函数调用一定出现错误，所以，这个outer的c不能释放

6.8 defer

defer意思是推迟、延迟。语法很简单，就在正常的语句前加上defer就可以了。在某函数中使用defer语句，会使得defer后跟的语句进行延迟处理，当该函数即将返回时，或发生panic时，defer后语句开始执行。注意os.Exit不是这两种情况，不会执行defer。

同一个函数可以有多个defer语句，依次加入调用栈中(LIFO)，函数返回或panic时，从栈顶依次执行defer后语句。执行的先后顺序和注册的顺序正好相反，也就是后注册的先执行。

defer后的语句必须是一个函数或方法的调用

7.7 深浅拷贝

在Go语言中，‌理解深浅拷贝的概念对于处理数据和内存管理至关重要。‌浅拷贝（‌Shallow Copy）‌和深拷贝（‌Deep Copy）‌是两种在复制数据结构时常用的策略，‌它们在处理复杂类型（‌如结构体、‌切片、‌映射等）‌时表现不同。‌

浅拷贝

浅拷贝仅复制一个对象的最外层，‌不会复制对象内部的子对象。‌在Go语言中，‌浅拷贝可以通过简单的赋值操作实现。‌对于切片、‌映射和通道等引用类型，‌赋值操作实际上只是复制了引用（‌即内存地址）‌，‌而不是它们指向的数据。‌

在上面的例子中，‌newSlice是slice的一个浅拷贝。‌修改newSlice的元素也会影响slice，‌因为它们指向相同的底层数组。‌

深拷贝

深拷贝会复制一个对象及其所有子对象，‌确保原对象和副本在内存中是完全独立的。‌在Go语言中，‌实现深拷贝通常需要手动复制对象的每个字段，‌或使用编码/解码的方式来实现。‌

在这个例子中，‌copy是original的一个深拷贝。‌修改copy的字段不会影响original。‌

总结

  • 浅拷贝仅复制对象的最外层，‌不复制内部子对象。‌
  • 深拷贝复制对象及其所有子对象，‌确保原对象和副本在内存中完全独立。‌
  • 在Go语言中，‌浅拷贝可以通过简单的赋值操作实现，‌而深拷贝通常需要更复杂的操作，‌如手动复制或使用编码/解码方式。‌
